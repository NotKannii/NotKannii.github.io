Question 1: What is the difference between non-semantic and semantic tags in HTML?

The difference between non-semantic and semantic tags in HTML is efficiency, practicallity, and meaning behind each tag.

Starting with non-semantic tags. This usually consists of <div> and <span> tags. Though these tags can be used in the same way as semantic tags, it's confusing to look at. Because all the tags are the same, it would be hard to tell what each line of text or code is meant to be.

Moving onto semantic tags, these tags are more efficient and can be better understood, even without the text. Semantic tags consist of tags like <header>, <aside>, <main>, and more. It not only makes the code more readable and easier to maintain but also enhances accessibility and SEO.
